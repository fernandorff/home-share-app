# QA Mobile — app inteiro (Playwright) + correção de bugs

**Data:** 2026-06-30 · **Escopo:** DEV apenas (sem commit/deploy até o usuário pedir)

Loop: testar o app inteiro no **viewport mobile (~390x844)** via Playwright,
rodar smoke + todos os testes disponíveis, **encontrar e corrigir bugs** até
ficar tudo perfeito. Releia este spec a cada iteração e valide **item a item**.

## Setup
- Dev server na porta 3000 (já rodando). Login: assinar um JWT de sessão do
  usuário id 1 com o `JWT_SECRET` do `.env` (64 chars) e injetar os cookies
  `bolitas_session` + `bolitas_group` via Playwright.
- Viewport mobile: 390x844. Testar nos dois temas (default e bolitas) ao menos
  por amostragem (cookie `bolitas_theme`).

## Detector automático de bug (rodar em CADA página)
1. **Sem overflow horizontal:** `document.documentElement.scrollWidth <=
   window.innerWidth + 1`. Se estourar, achar o elemento culpado e corrigir.
2. **Sem erros de console** (browser_console_messages onlyErrors): 0 erros.
3. **Bottom nav** (6 itens) visível e clicável; header não quebra.
4. Áreas de toque mínimas razoáveis; nada cortado/sobreposto.

## Páginas e fluxos (cada um no mobile)
1. **/auth/login** (deslogado — limpar cookies): form cabe, inputs usáveis, sem overflow.
2. **/despesas — Lista:** cards (não tabela), sem overflow, ⋯ abre menu, tocar
   abre o modal de detalhe; busca via Filtrar.
3. **/despesas — Por pessoa:** seletor de pessoa troca; itens em **cards**
   (modelo da lista), sem colunas espremidas.
4. **Modal Filtros:** abre, chips toggle multiselect, aplicar fecha + filtra,
   chips removíveis, Limpar; cabe na tela.
5. **Modal Nova despesa:** form cabe, chips de categoria/plataforma/forma, split
   igual/custom, salvar cria e recarrega.
6. **Modal Detalhe:** dimensões + divisão + Editar/Excluir; cabe.
7. **Modal Importar CSV:** abre e cabe.
8. **/saldos:** cards, "por pessoa", quem-paga-quem, gráficos por categoria/mês,
   modal registrar pagamento; sem overflow.
9. **/compras:** adicionar item, marcar comprado, limpar comprados.
10. **/catalogos:** 3 seções, abrir modal Adicionar, validação.
11. **/atividade:** feed carrega, diffs legíveis, sem overflow.
12. **/casa:** código de convite, copiar, moeda (select), membros, nova casa.
13. **Header menus:** casa selector, tema, idioma, usuário — abrem e cabem.
